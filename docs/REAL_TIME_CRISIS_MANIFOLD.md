# Real Time Crisis Manifold

The `Real Time Crisis Manifold` is the cyberdeck layout for urgent, changing situations.

## Purpose

- Keep the human oriented when the situation is moving fast.
- Put the most important controls and alerts in one live surface.
- Let the AI generate the right response blocks in context.
- Keep the interface terminal-skinned, compact, and readable under pressure.

## Layout Rules

- Alarm rail at the top for immediate status.
- Status core in the middle for current system state.
- Action strip for high-priority choices and confirmations.
- Memory log for the running conversation and relevant context.
- AI response blocks can appear inline as the situation changes.

## Behavior

- The manifold should reflow with the crisis state instead of staying fixed.
- The UI should privilege fast recognition over decoration.
- Important turns can be promoted into memory from the live hangout log.
- The layout should work as a real control surface, not just a transcript.

## Notes

- This is part of the Weyland-Yutani cyberdeck shell, not the harness source of truth.
- It sits on top of the live conversation and memory system.
